为了让 Web 前端项目在生产环境中得到最佳的性能和体验，工程师们在前端工程化领域推出了一系列的工具。

与此同时，还需要工程师花费大量的时间去熟悉和配置这些工具。但是，这些工具可能会让工程师们感到困惑，因为他们不知道他们应该使用哪一个。另外，工具的更新迭代过程中，工程师们又不得不去学习和使用它们，将新的实践应用到已有项目中。

## 把精力花在刀刃上

Next.js 为我们提供了一种以 React 框架为基准的、基于配置与共识的更加简单的方式去构建和部署 Web 应用。我们可以享受来自于 Vercel 以及他们的技术团队的支持，以很低的代价得到了 Web 开发的最佳实践。例如：

- React SSR/SSG
- [图片](https://nextjs.org/docs/basic-features/image-optimization)、[字体](https://nextjs.org/docs/basic-features/font-optimization)、[脚本](https://nextjs.org/docs/basic-features/script)等资源的优化

这些能力使得开发者从复杂和繁琐的配置中解脱出来，从而把精力花在业务开发上。
